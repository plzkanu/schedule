# IT 프로젝트 현황 및 일정관리 – GitHub + Replit 배포 가이드

## 흐름 요약

1. **로컬(Cursor)** 에서 코드 수정 후 **GitHub** 로 푸시  
2. **Replit Shell** 에서 GitHub 최신 코드로 동기화  
3. **Replit** 에서 Republish 로 재배포  

---

## 사전 준비 (최초 1회)

### Supabase DB 마이그레이션

배포 전 Supabase에 `it_*` 테이블이 있어야 합니다.

1. [Supabase Dashboard](https://supabase.com/dashboard) → 해당 프로젝트 → **SQL Editor**
2. 저장소의 `supabase/apply-all-migrations.sql` 내용을 붙여넣기 → **Run**
3. **Table Editor**에서 아래 테이블 생성 확인  
   - `it_projects`, `it_tasks`, `it_task_dependencies`, `it_activity_logs`, `it_users`
4. **Project Settings → API → Reload schema** (또는 1~2분 대기)

> 이미 마이그레이션을 적용한 DB라면, `apply-all-migrations.sql`의 `IF NOT EXISTS` / `ON CONFLICT` 구문 덕분에 중복 실행해도 대부분 안전합니다.

### RLS 오류 예방 (권장)

프로젝트·사용자 등록 시 `new row violates row-level security policy` 오류가 나면 SQL Editor에서 한 번 실행:

```sql
ALTER TABLE it_projects DISABLE ROW LEVEL SECURITY;
ALTER TABLE it_tasks DISABLE ROW LEVEL SECURITY;
ALTER TABLE it_task_dependencies DISABLE ROW LEVEL SECURITY;
ALTER TABLE it_activity_logs DISABLE ROW LEVEL SECURITY;
ALTER TABLE it_users DISABLE ROW LEVEL SECURITY;
NOTIFY pgrst, 'reload schema';
```

또는 `supabase/migrations/007_it_tables_disable_rls.sql` 내용을 실행합니다.

## 1단계: 로컬에서 코드 수정

Cursor에서 수정 후 저장합니다.

로컬 개발 서버:

```bash
npm install
npm run dev
```

- 기본 주소: http://localhost:3001  
- 로컬 환경 변수: 프로젝트 루트 `.env.local` (Git에 올리지 않음)

---

## 2단계: GitHub에 푸시

Cursor **터미널**에서:

```bash
git add .
git commit -m "변경 내용 설명"
git push origin main
```

- `main` 대신 기본 브랜치가 `master`라면 `git push origin master` 사용  
- 커밋 메시지는 실제 변경 내용으로 바꿔서 작성  
- `.env.local`, `.next` 폴더는 **커밋하지 않습니다**

---

## 3단계: Replit에서 동기화 (매번 배포 전 실행)

Replit 왼쪽 **Tools** → **Shell** 을 연 뒤, 아래를 **순서대로** 실행합니다.

### 3-1. 최신 코드 가져오기 (필수)

```bash
git fetch origin main
git reset --hard origin/main
```

- 브랜치가 `master`면 `main` 대신 `master` 로 바꿉니다.

### 3-2. 패키지 설치 (필수)

동기화 후 **항상** 실행하는 것을 권장합니다:

```bash
npm install
```

### 3-3. 프로덕션 빌드 (필수)

Next.js는 배포 전 빌드가 필요합니다:

```bash
npm run build
```

빌드가 성공해야 Republish 후 정상 동작합니다.

### 3-4. 한 번에 실행하려면

아래를 복사해 Shell에 붙여 넣고 실행:

```bash
git fetch origin main && git reset --hard origin/main && npm install && npm run build
```

---

## 4단계: 재배포

1. Replit 오른쪽 상단 **Deploy** 영역에서 **Republish** (또는 **Deploy**) 클릭  
2. 배포가 끝난 뒤 제공되는 URL로 접속해 동작 확인  
3. `/login` → 로그인 → `/dashboard`, `/projects` 화면 확인  

---

## Replit Secrets 설정

Replit **Tools** → **Secrets** 에 다음이 설정되어 있는지 확인하세요.

| Secret 이름 | 설명 |
|-------------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 프로젝트 URL (`https://xxxx.supabase.co`) |
| `SUPABASE_SERVICE_ROLE_KEY` | Service Role Key (Dashboard → Settings → API) |
| `AUTH_SECRET` | 세션 쿠키 서명용 랜덤 문자열 (`openssl rand -base64 32` 등으로 생성) |
| `SUPABASE_SSL_VERIFY` | (선택) 회사망 TLS 오류 시 `0` — 개발·내부망 한정 |

이 값들은 **GitHub에 올리지 말고**, Replit Secrets 에만 넣습니다.

### 로컬 `.env.local` 참고

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
AUTH_SECRET=your-random-secret
# SUPABASE_SSL_VERIFY=0
```

## Replit 실행 설정 (최초 1회)

Replit Deployment 설정에서 Run 명령이 아래와 유사한지 확인하세요.

| 항목 | 권장 값 |
|------|---------|
| Build | `npm run build` |
| Run | `npm run start` |

Replit이 `PORT` 환경 변수를 주는데 앱이 뜨지 않으면, `package.json`의 `start` 스크립트를 다음처럼 바꿉니다.

```json
"start": "next start -H 0.0.0.0 -p ${PORT:-3001}"
```

(Windows Replit Shell이 아닌 Linux Replit 기준. 변경 후 GitHub 푸시 → 3단계 동기화 → Republish)

## 반영이 안 될 때 체크리스트

| 확인 항목 | 조치 |
|-----------|------|
| Replit에서 `git status` 로 브랜치/파일 상태 확인 | `git fetch origin main && git reset --hard origin/main` 다시 실행 |
| 동기화 후 `npm install` / `npm run build` 안 함 | 3단계 전체 실행 후 Republish |
| Republish만 하고 Shell에서 동기화 안 함 | **3단계 전체** 실행 후 다시 Republish |
| Replit이 예전 커밋을 가리킴 | GitHub 푸시 성공 여부 확인 후, Replit Shell에서 `git log -1` 로 최신 커밋인지 확인 |
| 환경 변수 미설정 | Replit **Secrets** 3종(`NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `AUTH_SECRET`) 확인 후 Republish |
| `Could not find the table 'public.it_projects'` | Supabase SQL Editor에서 `apply-all-migrations.sql` 실행 |
| `new row violates row-level security policy` | 위 **RLS 오류 예방** SQL 실행 |
| Supabase 연결 실패 (fetch failed) | Secrets에 `SUPABASE_SSL_VERIFY=0` 추가 후 Republish (내부망 한정) |
| 빌드 실패 | Replit Shell에서 `npm run build` 로 오류 메시지 확인 후 수정·푸시 |

## Supabase 연결 참고

- 이 앱은 **Supabase Auth를 사용하지 않습니다.** 사용자는 `it_users` 테이블 + bcrypt 세션 쿠키로 인증합니다.
- Service Role Key는 **서버에서만** 사용합니다. 브라우저·GitHub에 노출하지 마세요.
- 입찰 시스템(`khnp_bid_*`)과 테이블이 겹치지 않도록 `it_*` 접두사를 사용합니다. **동일 Supabase 프로젝트를 공유해도 됩니다.**

## 요약: Replit에서 배포할 때마다

1. Shell 열기  
2. `git fetch origin main && git reset --hard origin/main && npm install && npm run build` 실행  
3. Republish 클릭  
4. 배포 URL에서 `/login` 동작 확인  

이 순서를 지키면 로컬에서 푸시한 내용이 Replit 배포에 반영됩니다.
